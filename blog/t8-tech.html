Week 8:

Skipper's log 4/19/2015:

Ruby on Rails, or Rails for short, is just that. It's a web application framework that runs the Ruby language and fast-tracks the development process by inserting basic assumptions to allow a programmer to cut down on the script they write while maximizing productivity within the code.

Rails is designed to pick what is, in its opinion, the best way to code. It makes basic assumptions using these opinions. By doing so, it uses conventional methodologies as the genesis of these assumptions. It also helps streamline the code so that you prevent against repetition that may create problems in the end product.

Rails is implemented as an extension software library to the Ruby language. The frame work it provides allows for the rapid developement of complex websites and web applications. In my opinion it's analgous to jQuery for JavaScript in the sense that it provides a lot of prepackaged functions and definitions that allow you to quickly accomplish what would take a great deal of time using only the core language itself.Rails is actually largely a Domain Specific Language (DSL) that has it's own directives seperate or distinct from the Ruby core. When using rail, the Ruby syntax language is used.
